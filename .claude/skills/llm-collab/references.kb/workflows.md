# Documentation Workflows

Documentation processes - when to document what, LLM consumption patterns, maintenance procedures, and setup checklists.

- When to read -- Planning documentation strategy, setting up new projects, understanding how LLMs navigate documentation, or establishing maintenance procedures.
- What's here -- Process-oriented guidance covering documentation lifecycle from creation through maintenance. Topics include setup checklists, maintenance procedures, decision frameworks for documentation scope, LLM interaction patterns, and when simpler documentation suffices.
- What's NOT here -- File format specifications (see file-types.d/), general principles (see guidelines.d/), or project-specific processes (those belong in the project).
- How to use -- Browse workflows.d/ when you need procedural guidance about documentation tasks or decision-making frameworks.
