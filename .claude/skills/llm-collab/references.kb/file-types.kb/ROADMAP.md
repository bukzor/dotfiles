# ROADMAP.md

**Purpose:** Bird's-eye view of project direction and major milestones

**Audience:** Contributors, stakeholders, users evaluating the project

**Location:** Project root (next to README.md)

**Must contain:**
- List of milestones with status
- Brief goal description for each
- Links to detailed milestone docs

**Structure:**
```markdown
# Roadmap

## Milestones

### Milestone 1: [Name]

**Status:** Not Started | In Progress | Complete

**Goal:** [What this milestone achieves]

See [docs/milestones/01-milestone-name.md](docs/milestones/01-milestone-name.md) for details.
```

**Status values:**
- `Not Started` - Planned but no work begun
- `In Progress` - Actively being worked on
- `Complete` - All success criteria met

**Relationship to milestones/:**
ROADMAP.md is the index/summary; docs/milestones/ contains full breakdowns.
Update ROADMAP.md when milestone status changes.

**When to update:**
- Starting work on a new milestone
- Completing a milestone
- Reprioritizing milestones
- Adding new planned milestones
