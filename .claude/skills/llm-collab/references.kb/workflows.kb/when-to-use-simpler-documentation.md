# When to Use Simpler Documentation

## Full Structure Works Well For

- Multi-session projects
- Complex systems with multiple subsystems
- Projects with LLM collaboration
- Projects with multiple contributors

## Consider Simpler Structure For

- Tiny projects (<500 lines)
- Single-session prototypes
- Pure exploratory work

## Core Principles (Keep These)

When deviating, preserve:
- **Separate user/contributor/LLM docs** - Different audiences need different info
- **Track current state explicitly** - What's done, what's next
- **Link with context, not just URLs** - Explain why to click
- **Tier detail levels** - Summary → deep dive
