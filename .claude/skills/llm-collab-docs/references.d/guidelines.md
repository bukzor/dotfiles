# Documentation Guidelines

Documentation quality principles - DRY practices, anti-patterns, linking conventions, and organization strategies.

- When to read -- Maintaining documentation quality, avoiding common pitfalls, ensuring consistency across projects, or deciding how to structure documentation.
- What's here -- Cross-cutting principles that apply to all documentation types. Topics include keeping docs DRY, linking conventions, when to use subdirectories, anti-patterns to avoid, and entry point design for common questions.
- What's NOT here -- Specific file type formats (see file-types.d/), step-by-step procedures (see workflows.d/), or schemas (see file-types.jsonschema.yaml).
- How to use -- Browse guidelines.d/ when you have a documentation design question or want to improve documentation maintainability.
