# File Type References

Documentation file type specifications - what each file type means, when to use it, required structure and conventions.

- When to read -- Creating or updating documentation files, need to know conventions for a specific file type (README, ADR, devlog, etc.).
- What's here -- Individual file type specifications with format requirements, audience descriptions, and usage guidelines. Each file explains one documentation file type pattern.
- What's NOT here -- General documentation principles (see guidelines.d/), workflow procedures (see workflows.d/), or project-specific documentation (those belong in the project).
- How to use -- Browse file-types.d/ to find the specific file type you're working with, or consult file-types.jsonschema.yaml for the formal schema.
