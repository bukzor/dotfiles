# Instruction Patterns

Reusable patterns for writing LLM-consumable procedures.

## What belongs here

Each file documents one pattern:
- The pattern (what to do)
- When to apply it
- Example from tdd-posthoc or elsewhere

## What does NOT belong

- tdd-posthoc-specific constraints (those go in constraints.kb/)
- Execution scenarios (those go in scenarios.kb/)
