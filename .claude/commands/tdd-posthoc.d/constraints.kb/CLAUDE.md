# Constraints

Boundaries and rules for tdd-posthoc, with rationale.

## What belongs here

Each file documents one constraint:
- The rule itself (terse)
- Why it exists
- What happens if violated
- How it was discovered/validated

## What does NOT belong

- The constraint list itself (that's in tdd-posthoc.md Principles)
- Implementation details
- Scenarios (those go in scenarios.kb/)
