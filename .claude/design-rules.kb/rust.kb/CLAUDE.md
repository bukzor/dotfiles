# Rust Design Rules

Rust-specific API design principles and known pitfalls.

## What belongs here

- Design rules specific to Rust's type system, ownership, or trait machinery
- Known limitations that affect API design decisions
- Patterns that bridge the gap between Rust and more familiar (Python) mental models

## What does NOT belong

- General design principles (those go in parent `design-rules.kb/`)
- Rust syntax tutorials
- Crate-specific implementation notes
