Refactor git-localhost-store to extract user-facing `git-restore-repo` command from hook-only implementation.

## What Happened

- Moved `bin/init` → `lib/init` (internal helper)
- Moved `template-repo/hooks/post-init` → `bin/git-restore-repo` (user-facing)
- Updated hooks (`post-index-change`, `pre-commit`) to call via XDG path
- Updated README.md with new structure and `git restore-repo` documentation
- Tested both automatic (hook-triggered) and manual invocation
- Tested recovery scenario (rm -rf → recreate → restore)
- Created ADR documenting the decision
- Updated CLAUDE.todo.md marking extraction complete

### Attempted

- Initially tried `HERE` pattern for hooks to find `git-restore-repo` via relative path
  - Failed: hooks are copied to each repo's `.git/hooks/`, so `../../` doesn't lead back to git-localhost-store
- Considered three approaches for hook→implementation path resolution

### Discovered

- Git template files are **copied** not symlinked, so any code in template gets duplicated
- This creates tension: relative paths fail, but absolute paths enable central updates
- XDG standard path provides acceptable "hardcoded" absolute path
- llm-collab-docs skill helper scripts (new-adr.sh, new-devlog.sh) needed fixes for non-interactive use

## Decisions Made

### Use XDG Standard Path in Hooks

**Rationale:** Hooks call `${XDG_DATA_HOME:-$HOME/.local/share}/git-localhost-store/bin/git-restore-repo`. This enables central updates (no stale copies) while maintaining single-step installation.

**Alternatives:**
- Relative paths (fail due to hook copying)
- PATH dependency (adds installation complexity)
- Template-embedded code (creates stale copies)

**Impact:** See ADR 2025-11-21-000

### Move bin/init to lib/init

**Rationale:** Separate user-facing commands (`bin/`) from internal helpers (`lib/`). `init` is never called by users directly.

**Impact:** Clearer organization, reduces confusion about what's meant to be invoked
